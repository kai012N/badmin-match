# v2.0 — UI 簡化重構

## 問題來源

實際使用後收到的回饋：

1. **介面太亂** — 資訊密度過高，手機操作困難
2. **手機跨 Tab 無法拖曳** — 選手在「等待區」Tab，場地在「排場」Tab，無法跨 Tab 拖曳
3. **配對警告太多** — 每一組幾乎都有警告，使用者開始忽略它
4. **沒有批次匯入** — 每次球聚要一個一個新增選手，非常耗時

---

## 解決方案

### 問題一：手機跨 Tab → Tap-to-Assign

不修改拖曳機制，改用「點選指派」流程：

```
點選等待區選手 chip → 選手被選中（藍框高亮）
↓
點擊場地空格 → 自動指派，清除選中狀態
```

新增 `selectedPlayerId` 到 GameContext，全域管理選中狀態。

### 問題二：Tab 數量縮減

從 5 個 Tab 縮減為 3 個：

```
舊：排場 | 等待區 | 推薦配對 | 選手 | 設定
新：🏸 排場 | 👥 選手 | ⚙️ 設定
```

排場 Tab 內整合等待區 + 推薦配對，一頁搞定。

### 問題三：配對警告收合

推薦配對的警告從逐條展示改為：

```
⚠ 3 項重複配對  ▼
```

點擊才展開詳細，預設收合。減少視覺干擾。

### 問題四：批次匯入

新增 `BatchImportPanel`，貼上文字名單一次匯入多人。

## 關鍵技術決策

### 為什麼不修拖曳、改用 tap 流程

手機的 touch 事件與 drag 事件衝突，@dnd-kit 的 TouchSensor 需要長按 250ms 才觸發，使用體驗不直覺。Tap-to-assign 更符合手機使用習慣。

### selectedPlayerId 不存 localStorage

這是 UI 暫存狀態（選中誰），不是業務資料，重新整理應該清除。用 `useState` 而非存入 reducer。
